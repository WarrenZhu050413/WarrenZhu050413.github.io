Media: How to Add

Different approaches for capturing media easily

Option 1: Markdown Files (Current)

Create a markdown file in _media/Easy

Manually create a .md file with frontmatter. Works with Jekyll's existing collection system.

---
title: "The Real Computer Revolution"
type: paper
url_link: "https://example.com/paper.pdf"
creator: Alan Kay
date: 2025-12-08
---

My notes about this paper...

Pros

  • No build tooling needed
  • Version controlled in git
  • Notes support markdown

Cons

  • Requires file system access
  • Manual date formatting
  • Need to remember frontmatter keys
Option 2: CLI Command

One-liner terminal commandEasy

Simple bash script to create media entries from command line.

# Add a paper
media add paper "The Mother of All Demos" "https://..." --creator "Doug Engelbart"

# Add a video
media add video "Inventing on Principle" "https://..." --creator "Bret Victor"

# Quick add (auto-detects type from URL)
media add "https://youtube.com/watch?v=..."
media add "https://arxiv.org/abs/..."
1Copy URL
2Run command
3Done!
Implementation idea: A simple shell script that creates the .md file with proper date and slug.
Option 3: Bookmarklet

Browser bookmarklet → copy frontmatterEasy

Click a bookmarklet on any page to copy the frontmatter to clipboard.

javascript:(function(){
  const title = document.title;
  const url = window.location.href;
  const type = url.includes('youtube') ? 'video' :
               url.includes('arxiv') ? 'paper' : 'link';
  const date = new Date().toISOString().split('T')[0];

  const frontmatter = `---
title: "${title}"
type: ${type}
url_link: "${url}"
date: ${date}
---

`;
  navigator.clipboard.writeText(frontmatter);
  alert('Copied to clipboard!');
})();
1Click bookmarklet
2Paste into new file
3Add notes
Option 4: Single Data File

All media in one YAML fileEasy

Instead of separate files, maintain a single _data/media.yml file.

# _data/media.yml
- title: "The Real Computer Revolution"
  type: paper
  url: "https://..."
  creator: Alan Kay
  date: 2025-12-08
  notes: |
    My notes here...

- title: "The Mother of All Demos"
  type: video
  url: "https://youtube.com/..."
  creator: Doug Engelbart
  date: 2025-12-07

Pros

  • One file to manage
  • Easy to reorder
  • Quick to add new entries

Cons

  • No individual permalink per item
  • Long notes get unwieldy
  • File grows over time
Option 5: Obsidian/Logseq Integration

Sync from PKM toolMedium

Tag items in Obsidian/Logseq and sync to Jekyll.

# In Obsidian, tag notes with #media
# A script syncs tagged notes to _media/

# obsidian-to-jekyll.py
for note in vault.notes_with_tag('media'):
    create_media_entry(
        title=note.title,
        url=note.get_property('url'),
        type=note.get_property('type'),
        notes=note.content
    )
Best for: If you already use Obsidian for note-taking
Option 6: Alfred/Raycast Workflow

Keyboard shortcut → formMedium

Trigger with hotkey, paste URL, hit enter.

Recommendation

Start with: CLI + Data FileEasy

Combine the simplicity of a single data file with a quick CLI command.

# Simple add command
media add "https://youtube.com/..." --note "Great talk on X"

# Appends to _data/media.yml:
- title: "Auto-fetched Title"
  type: video
  url: "https://youtube.com/..."
  date: 2025-12-08
  note: "Great talk on X"
1Copy URL
2Run media add [url]
3Done!
Why this works:
  • One command, minimum friction
  • Auto-fetches title from URL
  • Auto-detects type (YouTube → video, arXiv → paper)
  • Single file keeps everything organized
  • Easy to migrate to separate files later if needed
